Aquarium Manager API

Complete REST API for managing aquariums, inhabitants, accessories, and ornaments. Built with Java Spring Boot and secured with JWT authentication.

Quick Start

  1. Register or login to receive a JWT token
  2. Include token in Authorization header: Bearer <token>
  3. All responses use ApiResponse<T> wrapper format
https://web-production-8a8d.up.railway.app/api

Authentication

post /auth/register

Body: {firstName: string, lastName: string, email: string, password: string}

Response: {ownerId: number, token: string}

post /auth/login

Body: {email: string, password: string}

Response: {ownerId: number, token: string}

Aquariums

get /aquariums auth

Response: Aquarium[] - All user's aquariums

get /aquariums/{id}

Response: Aquarium - Basic aquarium info

get /aquariums/{id}/detail

Response: Aquarium - Full aquarium with collections

post /aquariums auth

Body: {name: string, length: number, width: number, height: number, substrate: SubstrateType, waterType: WaterType, color?: string, description?: string, state?: AquariumState}

Response: Aquarium

put /aquariums/{id} auth + owner

Body: Same as POST

Response: Aquarium

delete /aquariums/{id} auth + owner

Response: {aquariumId: number}

State History

get /aquariums/{id}/state-history

Response: StateHistory[]

get /aquariums/{id}/current-state-duration

Response: {aquariumId: number, currentStateDurationMinutes: number}

Inhabitants

Via Aquariums

get /aquariums/inhabitants auth

Response: Inhabitant[] - All user's inhabitants

get /aquariums/{aquariumId}/inhabitants

Response: Inhabitant[] - Aquarium's inhabitants

Standalone Resource

post /inhabitants auth

Body: {species: string, count: number, waterType: WaterType, type: string, aquariumId?: number, ...}

Response: Inhabitant

put /inhabitants/{id} auth + owner

Body: Same as POST

Response: Inhabitant

delete /inhabitants/{id} auth + owner

Response: {inhabitantId: number}

Accessories

get /aquariums/accessories auth

Response: Accessory[] - All user's accessories

post /accessories auth

Body: {model: string, serialNumber: string, type: string, aquariumId?: number, ...}

Response: Accessory

post /aquariums/{aquariumId}/accessories/{accessoryId} auth + owner

Body: {[key: string]: any} - Properties to update

Response: Aquarium - Updated aquarium

Ornaments

get /aquariums/ornaments auth

Response: Ornament[] - All user's ornaments

post /ornaments auth

Body: {name: string, material: string, aquariumId?: number, ...}

Response: Ornament

Development Notes

CORS Configuration
Configured for Vercel frontend (*.vercel.app) and localhost development.
Authentication Flow
  1. Register/Login → Receive JWT token
  2. Store token securely
  3. Include in header: Bearer <token>
Data Relationships
  • Aquarium ← Owner (1:many)
  • Aquarium ← Inhabitants/Accessories/Ornaments (1:many)
  • Water type compatibility enforced
