Get Started

    API Endpoint

        https://web-production-8a8d.up.railway.app/api/

The Aquarium Manager API provides comprehensive access to manage aquariums, their inhabitants, accessories, and ornaments. Built with Java Spring Boot and secured with JWT authentication, it offers a complete solution for aquarium management systems.

All API responses are wrapped in a standardized ApiResponse<T> format. To use this API, you need to authenticate and receive a JWT token which must be included in the Authorization header for protected endpoints.

Base URL

https://web-production-8a8d.up.railway.app/api/

Response Format

{
  "status": "success" | "error",
  "data": T | ErrorDetails,
  "timestamp": 1234567890123,
  "message": "Optional message"
}

Authentication

The API uses JWT (JSON Web Token) based authentication. You need to register or login to receive a token, then include it in the Authorization header for protected endpoints.

Register New User

# Register a new user
curl -X POST https://web-production-8a8d.up.railway.app/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "John",
    "lastName": "Doe", 
    "email": "john.doe@example.com",
    "password": "securePassword123"
  }'

REQUEST BODY

Field Type Description
firstName String User's first name
lastName String User's last name
email String Valid email address (unique)
password String Password (minimum requirements apply) 
Response:
{
  "status": "success",
  "data": {
    "ownerId": 1,
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  },
  "timestamp": 1701234567890
}

Login

# Login with existing credentials
curl -X POST https://web-production-8a8d.up.railway.app/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john.doe@example.com",
    "password": "securePassword123"
  }'

Using the Token

Include the JWT token in the Authorization header for all protected endpoints:

Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

Data Models

Enumerations

AquariumState

Represents the current state of an aquarium:

  • SETUP - Aquarium is being set up
  • RUNNING - Aquarium is operational
  • MAINTENANCE - Aquarium is under maintenance
  • INACTIVE - Aquarium is inactive

SubstrateType

  • SAND - Sand substrate
  • GRAVEL - Gravel substrate
  • SOIL - Soil substrate

WaterType

  • FRESH - Freshwater
  • SALT - Saltwater

Aquariums

Manage aquariums including creation, retrieval, updates, and state management.

Get All User Aquariums

curl -X GET https://web-production-8a8d.up.railway.app/api/aquariums \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

Authentication: Required

Returns: Array of user's aquariums (basic info)

Get Specific Aquarium

curl -X GET https://web-production-8a8d.up.railway.app/api/aquariums/1

Authentication: Not required

Returns: Basic aquarium information

Get Aquarium with Full Details

curl -X GET https://web-production-8a8d.up.railway.app/api/aquariums/1/detail

Authentication: Not required

Returns: Complete aquarium with all inhabitants, accessories, and ornaments

Create New Aquarium

curl -X POST https://web-production-8a8d.up.railway.app/api/aquariums \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Community Tank",
    "length": 120.0,
    "width": 50.0, 
    "height": 60.0,
    "substrate": "GRAVEL",
    "waterType": "FRESH",
    "state": "SETUP",
    "color": "#4A90E2",
    "description": "My new aquarium"
  }'

REQUEST PARAMETERS

Field Type Required Description
name String Yes Aquarium name
length Double Yes Length in cm
width Double Yes Width in cm
height Double Yes Height in cm
substrate SubstrateType Yes SAND, GRAVEL, or SOIL
waterType WaterType Yes FRESH or SALT
state AquariumState No Default: SETUP
color String No Hex color code
description String No Optional description

Update Aquarium

curl -X PUT https://web-production-8a8d.up.railway.app/api/aquariums/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Updated Community Tank",
    "description": "Updated description"
  }'

Authentication: Required (owner only)

Delete Aquarium

curl -X DELETE https://web-production-8a8d.up.railway.app/api/aquariums/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

Authentication: Required (owner only)

Error Handling

The Aquarium Manager API uses standard HTTP status codes and provides detailed error information in the ApiResponse format.

Error Response Format

{
  "status": "error",
  "data": {
    "status": 400,
    "error": "Validation Error",
    "message": "Invalid aquarium dimensions",
    "path": "/api/aquariums",
    "timestamp": 1701234567890
  },
  "timestamp": 1701234567890,
  "message": "Validation failed"
}

HTTP Status Codes

Status Code Meaning Common Causes
200 OK Request successful
201 Created Resource created successfully
400 Bad Request Validation errors, malformed JSON, invalid parameters
401 Unauthorized Missing, expired, or invalid JWT token
403 Forbidden User doesn't own the resource (insufficient permissions)
404 Not Found Resource doesn't exist or user doesn't have access
409 Conflict Resource already exists (e.g., duplicate email during registration)
500 Internal Server Error Unexpected server error, database connectivity issues

Common Error Types

Validation Errors

Occur when request data doesn't meet validation requirements:

  • Missing required fields
  • Invalid email format
  • Invalid enum values
  • Negative dimensions or counts
  • Water type compatibility issues

Authentication Errors

Issues with JWT tokens:

  • Token missing from Authorization header
  • Token expired
  • Invalid token signature
  • Malformed token

Authorization Errors

User permissions issues:

  • Trying to modify another user's resources
  • Insufficient permissions for requested action

Water Type Compatibility

The API enforces water type compatibility between aquariums and inhabitants:

  • Freshwater aquariums can only contain freshwater inhabitants
  • Saltwater aquariums can only contain saltwater inhabitants
  • Attempting to add incompatible inhabitants will result in a validation error

  • Water Compatibility: Ensure inhabitants match the aquarium's water type
